## 슬라이드 2. 연구 배경 & 선행연구 차별점 (1분 30초)

### PPT 내용

**선행연구 비교 표:**

| 선행연구 | 핵심 기여 | 본 연구의 확장 |
|---|---|---|
| Hochreiter & Schmidhuber (1997) | LSTM 아키텍처 제안 | 분위수 출력 확장 (방법론 기반) |
| Joshi (2020) | 미국 금리 스프레드 ARIMA/LSTM 비교 | RMSE만 비교 → **DM 검정 + 방향 정확도** |
| Saha et al. (2023) | Quantile-LSTM 프레임워크 | 이상 탐지 → **금리 변화량 예측에 적용** |
| Nunes et al. (2025) | LSTM-LagLasso 채권 금리 예측 | 점 예측 → **분위수 회귀로 불확실성 추정** |
| Gao & Hyndman (2025) | 무차익 신경망 금리 예측 | 가격 수준 → **일별 변화량(Δy) 예측** |
| Liu & Cheng (2026) | 분포 강건 앙상블 미국 국채 | 미국 국채 → **한국 국채 + 시차 누수 검증** |

**기존 연구 vs 본 연구 핵심 차이:**

| 구분 | 기존 연구 | 본 연구 |
|---|---|---|
| 예측 대상 | 금리 수준(level) 또는 월별 방향 | **일별 변화량(Δy)** |
| 출력 | 점 예측 1개 | **분위수 3개 (q05, q50, q95)** |
| 구간 보정 | 없음 | **CQR (Romano et al. 2019)** |
| 누수 검증 | 대부분 미검증 | **시차 누수 발견 & 수정** |
| 통계 검증 | RMSE 비교만 | **DM 검정 + Walk-forward 3-fold** |

### 대본
> 먼저 선행연구와의 차별점을 말씀드리겠습니다. 저희 방법론은 여러 선행연구에 기반합니다. LSTM은 Hochreiter가 1997년에 제안했고, 분위수 회귀의 신경망 적용은 Taylor가 2000년에, 예측 구간 보정인 CQR은 Romano 등이 2019년에 제안했습니다.
>
> 기존 채권 금리 예측 연구들과 비교하면 세 가지가 다릅니다. 첫째, 일별 변화량이라는 노이즈가 큰 문제를 다룹니다. 둘째, 분위수 회귀로 불확실성을 함께 추정합니다. 셋째, 프로젝트 중 데이터 누수를 직접 발견하고 수정했습니다. 이 부분은 뒤에서 자세히 설명드리겠습니다.

**핵심 메시지:**

### 대본
> 베이스라인으로 Naive와 ARIMA를 설정했습니다. Naive는 "내일도 변화 없다"고 예측하는 모델이고, ARIMA는 시계열 표준 모델입니다.

## 슬라이드 6. XGBoost 분위수 회귀 (1분)

### PPT 내용

**XGBoost + Pinball Loss → 분위수 3개 출력:**

| 구분 | 일반 XGBoost | 분위수 XGBoost |
|---|---|---|
| 손실 함수 | MSE (평균 오차 제곱) | **Pinball Loss** (비대칭 손실) |
| 출력 | 평균값 1개 | **q05, q50, q95 각각** |
| 모델 수 | 1개 | **3개** (분위수마다 따로 학습) |

**이미지 삽입:** `page6_7_pred_interval_comparison.png` 상단 (XGBoost raw 예측 구간)

→ 구간 폭 평균 11.6bp, 시장 변동성에 따라 소폭 변동

**CQR (Conformal Quantile Regression) 보정:**
- 모델이 출력한 구간이 실제로 90%를 커버하는지 보장 없음
- Cal 구간(2022년) 실제 오차로 구간 폭 보정: q05 − Q̂, q95 + Q̂
- 보정 후 Coverage 85.3% → 98.4% (fold3 기준, Q̂=6.6bp)
- 단, 구간이 24.8bp로 넓어지는 tradeoff 발생

### 대본
> XGBoost에 분위수 회귀를 적용했습니다. 핵심은 손실 함수입니다. Pinball Loss라는 비대칭 손실을 사용하면, q=0.05로 학습된 모델은 하한을, q=0.95로 학습된 모델은 상한을 예측합니다. 이렇게 3개 모델을 따로 학습하면 90% 예측 구간이 만들어집니다.
>
> 차트를 보시면 XGBoost의 raw 구간 폭이 평균 11.6bp로, 변동성이 높은 시점에서 약간 넓어지는 걸 볼 수 있습니다. 여기에 CQR이라는 후처리 보정을 적용하면 Coverage가 올라가지만, 구간이 24.8bp까지 넓어지는 tradeoff가 있습니다. 이 부분은 뒤에서 LSTM과 비교할 때 다시 다루겠습니다.

## 슬라이드 7. LSTM 분위수 회귀 (1분 30초)

### PPT 내용

**전체 모델 구조:**
```
입력: (batch, 30일 시퀀스, 14변수)
         ↓
   LSTM 2층 (hidden=64, dropout=0.3)
         ↓
   마지막 시점 hidden state
         ↓
   Linear 층 (64 → 3)
         ↓
출력: [q05, q50, q95]  ← 3개 동시 출력
```

**XGBoost와의 차이:**

| 구분 | XGBoost | LSTM |
|---|---|---|
| 분위수 방식 | 모델 **3개** 따로 학습 | 모델 **1개**가 3개 동시 출력 |
| 입력 형태 | 피처 벡터 1개 (162차원) | **30일 시계열** (30×14) |
| 시간 의존성 | lag 피처로 간접 표현 | 게이트로 직접 학습 |

**이미지 삽입:** `page6_7_pred_interval_comparison.png` 하단 (LSTM v2 예측 구간)

→ 구간 폭 평균 10.3bp — XGBoost(11.6bp)보다 타이트하고 변동이 더 큼

**v1 → v2 학습 여정:**

| 단계 | 내용 | 결과 |
|---|---|---|
| v1 (8개 변수) | 5×5 grid search (25개 조합) | **전부 dir_acc ≈ 49.5%** |
| 원인 분석 | 누수 수정 후 미국 변수 당일 정보 차단 → 가장 강력한 신호 소실 | ACF 백색잡음, 관계 자동학습 한계 |
| v2 (14개 변수) | 파생변수 5개 추가 (spread, delta, crisis) | **3-fold 평균 59.2%, fold3 66.1%** |

> 포인트: "LSTM이 실패했다"가 아니라 **원인 분석 → 파생변수로 회복시킨 과정**이 프로젝트의 가치

### 대본
> LSTM 분위수 회귀입니다. 구조는 30일치 14개 변수 시계열을 LSTM 2층에 통과시켜 압축하고, Linear 층에서 q05, q50, q95를 동시에 출력합니다. XGBoost가 모델 3개를 따로 학습하는 것과 달리, LSTM은 모델 1개가 세 분위수를 동시에 학습합니다. 분위수를 만드는 건 모델 구조가 아니라 Pinball Loss입니다.
>
> 차트를 보시면 LSTM의 예측 구간이 XGBoost보다 좁고, 시장 상황에 따라 폭이 더 많이 변합니다. 평균 10.3bp로 XGBoost의 11.6bp보다 타이트합니다.
>

## 슬라이드 10. EMH — 61%가 왜 의미 있는가 (45초)

### PPT 내용

**효율적 시장 가설 (Efficient Market Hypothesis):**
- 모든 공개 정보가 이미 가격에 반영 → 누구도 50%보다 의미 있게 잘 맞히기 어려움
- 학술 논문 합격선: **52~53%만 통계적으로 유의해도 인정**
- 헤지펀드가 ML에 거액을 투자하는 이유 = **2~3%p의 우위**

**본 연구 결과:**
- XGBoost v2: **61.0%** + DM 검정 p < 0.001 → 우연이 아님을 통계적으로 입증
- LSTM v2: **59.2%** + DM 검정 p < 0.001 → 마찬가지로 통계 유의
- 두 모델 모두 Naive 대비 **+9~11%p** 우위
- Bootstrap 95% CI 하한이 학술 합격선 53%를 명확히 상회

### 대본
> 결과를 보시기 전에, 61%라는 숫자의 의미를 먼저 말씀드리겠습니다. 효율적 시장 가설에 따르면, 공개 정보가 이미 가격에 반영되어 있어서 누구도 50%보다 의미 있게 잘 맞히기 어렵습니다. 학술 논문에서는 52~53%만 유의해도 인정됩니다. 헤지펀드가 ML에 거액을 투자하는 이유가 고작 2~3%p의 우위 때문입니다.
>
> 저희 XGBoost v2는 61.0%, LSTM v2는 59.2%이고, 두 모델 모두 DM 검정에서 p < 0.001로 우연이 아님을 확인했습니다. 50% 기준선 대비 9~11%p 우위는 금융 시계열에서 상당히 의미 있는 수치입니다.

## 슬라이드 12. 오류 분석 (45초)

### PPT 내용

**모델은 언제, 왜 틀리는가?**

| 분석 축 | 내용 | 발견 |
|---|---|---|
| 방향성 오답 | 부호를 틀린 사례의 공통점 | 금리 모멘텀 전환점에 집중 |
| 큰 변동 미예측 | \|Δy\|>5bp인데 모델은 1bp 이내 예측 | FOMC, 한은 금리 결정일에 발생 |
| Coverage Miss | 90% 구간 이탈 시점 분석 | fold2(금리 인상기) XGB 70.1%, LSTM 66.0%로 급락 |
| fold1 vs fold3 | COVID vs 안정기 비교 | 위기 시 DM 검정 tie → distribution shift |

**핵심 결론:**
> 모델은 안정적 시장에서 유효하나, 레짐 전환(regime shift) 구간에서 한계.
> 이는 학습 데이터에 없는 패턴이 등장하는 본질적 문제.

**향후 대응 방향:**
- ACI (Adaptive Conformal Inference) — distribution shift에 적응하는 보정
- Regime detection 모델 → 위기 시 보수적 전략으로 전환

### 대본
> 오류 분석입니다. 모델이 주로 틀리는 시점을 분석하면, 금리 모멘텀이 전환되는 지점에 오답이 집중됩니다. 또 FOMC나 한은 금리 결정일 같은 이벤트에서 5bp 이상 큰 변동이 발생하는데, 모델은 이걸 예측하지 못합니다.
>
> 가장 뚜렷한 패턴은 fold2, 2021-22년 금리 인상기에서 Coverage가 XGBoost 70.1%, LSTM 66.0%로 급락한 것입니다. 학습 데이터에 없던 금리 인상이라는 새로운 레짐이 나타난 distribution shift 문제입니다. 이건 모든 ML 모델의 본질적 한계이고, 기말에서 ACI 같은 적응적 보정을 검토할 계획입니다.

## 예상 질문 & 답변

### Q1. "왜 LSTM이 아니라 XGBoost가 메인 모델인가?"
> XGBoost v2가 3-fold 평균 61.0% ± 2.8%로 LSTM v2의 59.2% ± 6.9%보다 안정적입니다. XGBoost는 시드 무관한 결정적 학습이고, LSTM은 시드에 따라 변동이 큽니다. 다만 RMSE에서는 LSTM이 약간 우세해서, 기말에 LSTM에 CQR을 적용하면 역전 가능성도 있습니다.

### Q2. "LSTM에 CQR은 왜 안 했나?"
> 시간 관계상 XGBoost에만 적용했습니다. LSTM raw Coverage가 77.3%인데, CQR을 적용하면 90%에 가깝게 올라갈 수 있고 구간도 XGBoost보다 좁을 수 있습니다. 기말 과제로 진행할 계획입니다.

### Q3. "XGBoost CQR 구간이 너무 넓은 것 같은데?"
> 맞습니다. fold3 기준 Q̂=6.6bp가 양쪽에 더해져서 raw 11.6bp → CQR 24.8bp로 2배 넓어집니다. Coverage는 98.4%로 과보장이 됩니다. 이건 Cal 구간(2022년)이 Test(2023-2025)와 변동성 패턴이 달라서 보정값이 과도하게 잡힌 것입니다. ACI(Adaptive Conformal Inference)로 개선 가능합니다.

### Q4. "fold1(COVID)에서 왜 두 모델 다 안 좋은가?"
> fold1은 train이 2010-2017(안정기)인데 test가 2020(COVID)입니다. 학습 데이터에 없던 패턴이 나타나는 distribution shift 문제입니다. 두 모델 모두 DM 검정에서 Naive와 tie — 이건 ML의 본질적 한계이고, regime detection으로 대응하는 것이 기말 과제입니다.

### Q5. "왜 환율을 안 썼나?"
> 한국 원/달러 환율은 정부 개입이 강합니다 (외환당국 방어선, 국민연금 외환스왑 등). 학습하면 거시 신호가 아닌 정책 신호를 학습하게 됩니다. DXY(달러 인덱스)가 EM 자본유출 채널을 대체합니다.

### Q6. "파생변수가 사후 끼워맞추기 아닌가?"
> 세 가지 근거가 있습니다. 첫째, 모든 변수가 도메인 논문에 근거합니다. 둘째, 변수 정의를 결과 확인 전에 사전 기록했습니다. 셋째, Walk-forward 3-fold에서 fold1(52%), fold2(59%), fold3(64-66%)로 일관된 개선을 보여 과적합이 아님을 확인했습니다.

### Q7. "Coverage와 Sharpness 중 뭐가 더 중요한가?"
> tradeoff입니다. Coverage가 높으면 "구간 안에 실제값이 들어올 확률"이 높지만, 구간이 너무 넓으면(Sharpness 큼) 실용적 가치가 떨어집니다. 이상적으로는 Coverage 90%를 유지하면서 Sharpness를 최소화하는 것이 목표입니다. 현재 XGBoost CQR은 Coverage 87.3%에 Sharpness 15.64bp이고, LSTM raw는 Coverage 77.3%에 Sharpness 9.50bp입니다. LSTM에 CQR을 적용하면 Coverage를 올리면서도 구간을 좁게 유지할 수 있을 것으로 기대합니다.
