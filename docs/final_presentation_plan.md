## 0. 한 줄 요약

중간에서 우리는 변수 선택을 **단일 정량 필터(상관 / VIF<10 / Granger)** 로 22개 → 8개로 **기계적으로 제거**했다.
교수님 피드백의 핵심은 *"VIF 같은 수치만으로 제거하지 말고, 도메인으로 그룹핑한 뒤 예측·해석 목적에 따라 다르게 선택하라"* 이다.
**기말의 핵심 개선 = 변수 선택을 「도메인 그룹핑 + 예측/해석 이원화」 체계로 재설계**하고, 이를 기존 3대 기술 과제(예측구간·위기앙상블·백테스트)와 함께 제시한다.

### 0.1 ⭐ 프로젝트 우선순위 — 예측이 1순위, 해석은 후순위

본 프로젝트의 목표는 **예측 성능**이다(해석은 프로젝트 목표가 아님). 교수님도 *"예측을 먼저 하면 변수를 빼든 더하든 문제가 안 된다"*고 명시 → 예측 우선은 피드백과 **정합적**.

| Tier | 항목 | 노트북 | 비고 |
|---|---|---|---|
| **T1 (필수)** | 예측 파이프라인: 변수 재설계 + **성능 분포 정당화(⑥)** + v3 결과/모델비교 + 예측구간(§3.1) | 09, 11, 12 | 프로젝트 본체 |
| **T2 (권장)** | 트레이딩 백테스트 (예측의 실용성 입증) | 13 | 정직 alpha |
| **T3 (선택, 시간 시)** | **최종 모델 SHAP 해석** | 10 | 시간 부족 시 **생략 가능** |

> **변수 선택 시의 해석은 유지**(도메인 그룹핑 — F1·F3): 단 이건 *선택의 근거*용 경량 해석이고, 별도의 **SHAP 분산 deep-dive(노트북 10)는 Tier 3**.
> **핵심 전환**: 변수 선택 정당화를 SHAP(해석·분산 위험)이 아니라 **성능 분포(⑥, 예측 지향)**로 한다 → 교수님이 경고한 *SHAP 기여 분산(F2)*을 **피하는 방법**이 곧 우리의 서사.
> ✅ **결정 (b) SHAP 최소 유지**: 차별점 #3은 'TreeSHAP 해석 + 성능 분포 기반 변수선택' **둘 다** 제시. 발표 1~2장은 **중간 산출물 재사용**(`shap_group_importance.png`·`shap_beeswarm_top15.png`)으로 충당 → 신규 작업 거의 없음. 신규 SHAP 분산 deep-dive(노트북 10)는 Tier 3 유지.

## 1. 교수님 중간 피드백 (원문 + 해석)

> **원문 요지**
> - 회귀/XGBoost에서 VIF(다중공선성)가 높다고 변수를 **무조건 제거하는 것은 지양**.
> - 특정 변수가 **먼저 진입(분기)하면 타 변수의 SHAP 기여도를 분산**시킬 수 있다.
> - 단순 수치 기반 제거가 아니라 **도메인 지식으로 변수를 그룹핑**하여 의미 있는 변수를 선택.
> - **예측 vs 해석** 중 어디에 방점을 두느냐에 따라 변수 선택 기법을 **다르게** 접근.

| # | 피드백 | 중간 발표에서 우리가 한 것 | 왜 문제인가 | 기말 대응 (요약) |
|---|---|---|---|---|
| F1 | VIF 높다고 무조건 제거 금지 | VIF<10 hard cut으로 국내 5Y·1Y 등 제거(슬라이드 5) | 공선성 변수에도 고유 신호가 있고, 트리·정규화 모델은 공선성에 강건 → 사전 제거는 정보 손실 | **예측 모델엔 사전 VIF 제거 미적용**, 정규화/CV로 선택 (2.3) |
| F2 | 먼저 진입한 변수가 타 변수 SHAP 분산 | grouped SHAP(13그룹 합산)까지만 함 | Δus10y(t-1)가 상관 높은 파트너의 기여를 흡수 → 개별 SHAP 과대/과소 | **상관 군집 기반 그룹 SHAP + SHAP interaction + permutation/drop-column 교차검증** (2.4) |
| F3 | 도메인 그룹핑으로 의미 있는 변수 선택 | 통계 점수(3·2점)로 기계 선택 | 경제적 의미가 점수에 가려짐 | **5개 경제 테마로 그룹핑 후 대표 변수 선정** (2.2) |
| F4 | 예측 vs 해석 목적별 차등 | 한 세트(8변수)로 예측·해석 겸용 | 두 목적의 최적 변수 집합이 다름 | **예측용/해석용 파이프라인 이원화** (2.3–2.4) |

> 📌 **교수님 전사본 추가 뉘앙스** (위 표 보강)
> - **VIF는 제거 임계가 아니라 "해석의 문제가 있다"는 경고 신호.** 확 빼지 말고 → (i) 다른 변수 선택 기법을 쓰거나, (ii) VIF 높은 변수들의 그룹핑 성격을 보고 **그 중 가장 의미 있는 것**을 도메인으로 골라 정리.
> - **목적이 예측이면 변수 가감은 상대적으로 자유롭다** (교수님 명시). 반면 해석은 막 빼면 *남은 변수가 진짜 중요한 게 아니라 빼버려서 나아 보이는 것*일 수 있어 위험. → 본 프로젝트는 **성능이 목적**이므로 변수 가감의 자유도를 활용하되, 그 선택이 타당함을 **성능 분포로 입증**(§2.6 ⑥).

## 2. 핵심 개선: 변수 선택 방법론 재설계

### 2.1 중간 방식의 한계 (자기비판)

- 3단계 필터(|r|>0.05 / VIF<10 / Granger p<0.05)는 **해석 친화적**이지만 **예측 최적**과는 다른 목적함수.
- VIF hard cut은 "공선성=나쁨"이라는 가정에 의존 → 트리 부스팅/규제 회귀에서는 성립하지 않음.
- 결과적으로 **예측 성능에 기여할 수 있는 변수를 사전에 버렸을 가능성** 존재.

### 2.2 도메인 기반 그룹핑 (F3)

22개 후보 → 통계 점수가 아니라 **경제 채널(테마)** 로 먼저 묶는다.

5개 경제 채널(테마)로 묶는다. **변수 배정 확정안은 §2.6 ①** 참조(이 표는 개념 소개).

| 그룹 | 경제적 의미 | 핵심 변수 | 그룹핑이 주는 이점 |
|---|---|---|---|
| G1. 글로벌 금리 채널 | 미국 금리 레벨·모멘텀·정책·기대인플레 → 한국채 스필오버 | us_treasury_10y, delta_us10y_t1, us_fed_funds, us_breakeven_10y | 채널의 핵심 동인, SHAP 1위 그룹 |

### 2.3 예측용(Prediction) 파이프라인 — F1·F4

목표: **방향 정확도 / 구간 품질 최대화**. 공선성 사전 제거 ❌.

- 후보를 **넓게 유지**(그룹 내 변수 다수 포함)하고, 선택을 모델·CV에 위임.
- 기법:
  - XGBoost 내장 규제(L1/L2, `reg_alpha`/`reg_lambda`) + early stopping.
  - **Walk-forward CV 기반 선택**: RFECV 또는 permutation importance(검증셋)로 변수 집합 결정.
  - (옵션) 그룹 단위 정규화: group lasso 성격의 변수 묶음 선택.
- 산출: "VIF 사전 제거 vs 미제거" **ablation 표** — 미제거가 예측에서 동등/우수함을 수치로 입증.

### 2.4 해석용(Interpretation) 파이프라인 — F2·F4 〔🟡 Tier 3 · 시간 시만〕

> 예측이 1순위이므로 이 절은 **선택**. 변수 선택 정당화는 §2.6 ⑥(성능 분포)로 대체하고, 아래 SHAP 분산 분석은 시간 여력이 있을 때만 수행.

목표: **SHAP 기여도의 안정적·정직한 해석**. 여기서는 **절약(parsimony)·저공선성**이 미덕.

- **SHAP 기여 분산(F2) 대응 3종 세트**:
  1. **상관 군집 기반 그룹 SHAP**: 피처를 상관(또는 SHAP correlation)으로 hierarchical clustering → 군집 단위로 SHAP 합산. (중간의 "원본 변수별 합산"을 군집 단위로 확장)
  2. **SHAP interaction values**: Δus10y(t-1)와 한미금리차 등 상관 쌍의 상호작용/중복 정량화.
  3. **교차 중요도 비교**: TreeSHAP vs permutation importance vs drop-column importance — 공선성 하에서 서로 다르게 거동함을 보이고, **단일 지표 맹신의 위험**을 설명.
- 결과: "먼저 진입한 변수가 SHAP을 독식"하는 현상을 **시각적으로 보여주고**, 군집 SHAP으로 보정한 해석 제시.

**① 도메인 5개 그룹 — 실제 변수 배정 (F3)**

| 그룹 | 경제 채널 | 배정 변수 | 대표(1차안) |
|---|---|---|---|
| G1. 글로벌 금리 | 미국 금리 레벨·모멘텀·정책·기대인플레 | us_treasury_10y, delta_us10y_t1, us_fed_funds, **us_breakeven_10y** | **delta_us10y_t1** (그룹SHAP 1위) |

> - `us_breakeven_10y`(기대인플레=명목−실질)는 **명목 미국채에서 파생** → 미국 금리와 동조하므로 **G1**에 배정. G5는 `crisis_dummy` 단독(개념 일관성).
> - `kospi`는 **VIF가 아니라 Granger p=0.57로 탈락**(VIF 1.11 정상). 트리는 비선형을 잡으므로 선형 Granger 탈락을 재검증할 가치 → **최종셋 복원 X, ablation/히스토그램(②B·⑥) 테스트 케이스로만** 포함. 성능이 안 오르면 "탈락을 성능 증거로 재확인", 오르면 복원 재고.

**② F1 ablation 비교 단위** — walk-forward 3-fold로 3개 변수집합 비교

- (A) 중간 8변수 = VIF<10 + Granger 필터 통과분 (baseline)
- (B) **VIF 미적용 확장셋** = G2/G3 공선 변수(kr 5Y/1Y 등) + kospi 복원
- (C) **그룹 대표 + 규제** = 각 그룹 대표 변수 + XGBoost `reg_alpha`/`reg_lambda`
- 지표: dir acc · RMSE · DM p · interval score → **가설: (B)/(C) ≥ (A)** 면 VIF 사전 제거가 예측엔 손해였음 입증

**③ F2 군집 SHAP 클러스터링**

- 거리 = `1 − |피처 간 피어슨 r|` (보조: SHAP value correlation), hierarchical(average linkage)
- 절단 = |r|≥0.7 묶음 (dendrogram 시각 절단 병기)
- 군집 단위 |SHAP| 합산 → 중간의 "원본변수별 합산"을 **데이터 기반 군집**으로 대체

**④ F2 SHAP interaction 지목 쌍** (분산 시연용)

- `delta_us10y_t1 ↔ spread_10y_t1` (둘 다 미국금리 기반 — 분산 핵심 사례)
- `vix ↔ sp500` (r=−0.734, 확인된 강상관)
- `us_treasury_10y ↔ delta_us10y_t1` (레벨 vs 변화량)

**⑤ 파이프라인별 성공 기준 (F4)**

- **예측**: walk-forward dir acc ≥ v2(0.6163) **AND** DM p<0.05(vs Naive) 유지 **AND** interval score ≤ v2
- **해석**: 군집 SHAP에서 G1(글로벌 금리) 최상위 유지 + interaction으로 분산 설명 + 중요도 3종(TreeSHAP/permutation/drop-column) 순위 Spearman 상관 보고

**⑥ 변수 선택 타당성의 분포적 입증 (성능 목적 — 팀 제안)**

> "목적이 예측"이므로 변수 선택을 **도메인 근거 + 성능 분포**로 이중 입증. 선택되지 않은 변수·조합도 돌려보고 우리 선택이 분포의 **우측 꼬리(상위 백분위)**에 있음을 시각화 → 교수님 "잘 선정했다는 걸 비교·입증" 요구에 직접 응답.

- **(a) 랜덤 서브셋 벤치마크 분포**: 전체 후보 풀에서 동일 크기 부분집합을 N=300~500개 무작위 샘플 → 각각 walk-forward dir acc(또는 RMSE) 측정 → **히스토그램**. 도메인 선택셋 위치를 수직선·백분위로 표시.
- **(b) 변수별 한계 기여 히스토그램**: leave-one-out(선택셋에서 1개 제거 시 성능 하락) + add-one-in(제외 변수 추가 시 변화) → 각 변수의 marginal value. 제외 변수가 성능을 안 올린다는 **직접 증거**.
- ⚠️ **정직성 가드 1**: 이 분포는 **선택의 사후 정당화·검증용**이지 *선택 도구가 아님*. 최고 성능 서브셋을 골라 쓰면 data snooping (`experiments/` Path A 메타이슈와 동일). 도메인 선택을 1차로, 히스토그램은 보조 증거로만.
- ⚠️ **정직성 가드 2 (비용)**: 분포 sweep은 **경량 프록시**(단일 XGBoost 점예측, CQR/분위수 생략)로 돌리고, 최종 선택셋만 full 파이프라인 재검증.
- 산출: `09_feature_redesign.ipynb` 후반 + `scripts/`에 sweep 스크립트.

### 3.1 T1 상세 — 예측구간 축소 + 방향 결합 (팀 보강)

**문제의 3겹 분해**
- (a) **XGBoost 독립성**: q05/q50/q95를 **독립 모델 3개**로 학습 → 분위수 교차(q05>q50) 위험 + 세 경계가 서로·점예측과 무관하게 따로 움직임. (LSTM 공동출력은 덜 발생 → 비교 포인트)
- (b) **대칭 CQR 과보정**: raw coverage 85.3%(목표 90% 미달)를 단일 대칭 Q̂로 98.4%까지 끌어올리며 폭 11.6→24.8bp.
- (c) **방향과 분리**: 구간이 0을 항상 포함 → "오를지/내릴지" 정보 없음 = 실용성 부재.

**접근 (4갈래 + 평가 1)**
1. **분위수 비교차 제약** — post-hoc rearrangement(Chernozhukov et al.) 또는 단조 제약으로 q05≤q50≤q95 보장 → (a) 해결.
2. **비대칭 CQR (AsymVar)** — 단일 대칭 Q̂ 대신 상·하한 conformity score 각각의 분위수로 **보정량 독립 산정**(Romano et al. 2019 asymmetric variant) → 한쪽이 잘 보정되면 그쪽은 안 넓힘 → (b) 폭 절감.
3. **국소 적응 가변폭** — conformity score를 국소 변동성(rolling vol/VIX)으로 정규화 → **위기엔 넓고 안정기엔 좁은** 구간 자동 생성.
4. **raw 분위수 개선** — v3 변수·튜닝으로 raw coverage 90% 근접 → CQR 보정폭 자체 축소 → (b) 근본 원인 완화.
5. **방향 결합 평가 (핵심)** — 폭만 보지 말고 **"구간이 0을 배제하는 비율"**(=신뢰 방향 신호) + 부호/레짐별 conditional coverage를 지표화. 좁아진 구간이 한쪽(>0 또는 <0)에 온전히 위치하면 그때 방향 정보를 담음 → (c) "방향 잡으면 범위도 바뀐다"의 정량화.

> 시작점: 중간 준비 산출물 `quantile_v3_asymmetry*.png`, `interval_asymmetry_analysis.png`, `scripts/29_quantile_improvement.py`, `reports/no_leak_v2/quantile_v3_comparison.csv` 위에 확장. 핵심 메시지: *"현재 구간은 0을 항상 포함해 무의미. 비대칭·국소적응 CQR + 비교차 제약으로 좁히면, 좁아진 구간이 방향 신호 자체를 담는다."*

**왜 지금 구간이 "직선"처럼 보이나 + 가변폭이 타당한가**

- **원인 2개**: (1) 일별 금리변화는 노이즈·평균0이라 raw 분위수 예측이 거의 등분산(homoscedastic) → q05/q95가 q50의 ~일정 오프셋. (2) **대칭 CQR은 단일 스칼라 Q̂를 양쪽에 동일 가산** → 폭이 구조적으로 상수. → 현재의 평탄한 밴드는 버그가 아니라 대칭 CQR의 당연한 귀결.
- **가변폭·비대칭은 표준 기법이며 허용됨** (locally-adaptive conformal / normalized nonconformity = Papadopoulos 등; asymmetric CQR = Romano et al. 2019). 단 3조건:
  1. **누수 금지**: 폭은 **예측 시점 가용 정보**(rolling vol·VIX·레짐)에만 의존. 미래 실현값으로 폭 결정 시 누수 — 정직성 서사 직결. 캘리브레이션은 held-out.
  2. **마진 커버리지 90% 유지, 개선 대상은 조건부 커버리지**. 분포무가정에서 *정확한* 조건부 커버리지는 이론적 불가(Vovk; Lei & Wasserman) → **"근사적 조건부 커버리지 개선"**으로 정직하게 주장.
  3. ⚠️ **0 배제를 인위 강제 금지** — 억지로 한쪽으로 밀면 커버리지 붕괴. 비대칭은 데이터/분위수에서 **자연 발생**해야 하고, 그 결과 0을 배제하면 그게 진짜 방향 신호.
- **사전 진단 (노트북 12 첫 셀)**: 현재 구간폭의 **시계열**을 그려 실현 변동성과 상관 확인. 2020(코로나)·2022(인상기) 고변동기에도 폭이 평탄하면 → "직선" 가설 확증 + 가변폭 동기 입증. **기대치**: 일별 wiggle이 아니라 **레짐별(위기↔안정) 확대/축소**가 정상(과한 변동은 과적합 의심).

### 3.2 T3.1 — 2026 라이브 OOS 포워드 검증 (팀 보강)

**동기**: 현재 성능평가는 2023–2025 test(**종가 기준 체결 가정**)에 한정. 개발 중 test 반복 참조 가능성 + 종가 체결의 비현실성을 보완하기 위해, **학습에 전혀 안 쓰인 2026년 최근 ~3개월(약 60거래일)** 데이터를 신규 수집해 **동결된 v3 모델로 순수 포워드 예측 + 실제 수수료 반영 시뮬레이션**.

- **진짜 OOS**: 모델·CQR·임계값 전부 ≤2025로 **동결** 후 2026 데이터는 예측에만 사용(재학습 금지) → 누수 0.
- **빈도 = 일별 고정**: 모델 타겟이 *익일 일별 변화량(Δy)*, 입력도 일별 종가 → 빈도는 일별로 고정(인트라데이 불가·무의미). "실시간"=매일 최신 종가 갱신.
- **체결 = next-open (정직 버전)**: 신호는 **T 종가** 데이터로 산출 → 체결은 **T+1 시초가**(실제 체결 가능 가격). close-to-close(종가 체결)는 못 사는 가격 가정이라 낙관적 → next-open으로 "종가 기준" 비현실성 해소.
- **방향↔포지션 부호**: 모델은 *금리(yield)* 예측, 채권/ETF 가격은 **역방향** → **금리 상승 예측=숏, 금리 하락 예측=롱**. 신호 생성 시 이 부호를 명시(실수 방지).
- **데이터 수집**: 14개 입력변수 2026 Q1–Q2 — US(FRED: 10Y·fed funds·breakeven·DXY·S&P500·VIX), KR(ECOS: KTB 3Y·기준금리), 주가(yfinance). 모두 **일별만 제공**(시간 단위 데이터 없음), `scripts/02_collect_data.py` 출처 재사용. **1단계 = 2026 수집 가용성·파이프라인 호환 확인**.
- **실제 비용(기본안)**: `experiments/` Path C 프레임 재사용 — **KOSEF 국고채 ETF** 기준 현실 수수료·스프레드 (1bp/3bp 시나리오).
- ⚠️ **정직성 가드 (필수)**: ~60거래일은 검정력 약함(DM·Sharpe CI 넓음) → **"통계적 입증"이 아니라 "라이브 sanity-check / 방향성 확인"**으로 프레이밍. Path A 소표본·저참여 자기비판 기준 동일.
- **거래수단 = 국고채 ETF(KOSEF 국고채10년) 기본** — 수수료·운용보수 명확(`experiments/` Path C 재사용). KTB 선물은 대안(비용구조·접근 복잡).
- **데모 앱 연동** (`app/streamlit_app.py` 확장): ① 라이브 일별 예측 + 예측구간(§3.1), ② **수수료 조절 슬라이더 → 누적손익 실시간 반응** 패널 → *"거래비용에서 alpha 소멸"* 정직 서사를 시각화.
- **산출**: 2026 예측 vs 실현 비교, 라이브 dir acc, **next-open 체결·수수료 반영** 누적손익, 구간 coverage(§3.1 적용).

## 5. 발표 구성안 (변경점 중심)

**전제: 동일 청중(교수님·동료).** → 중간에 이미 발표한 내용(선행연구·베이스라인·모델 구조·누수 발견·최종모델 SHAP)은 **반복하지 않는다.** 중간 요약은 **1장 브릿지**로 압축하고, 10분 전부를 **기말 신규 작업**에 할애. 브릿지는 *"교수님 피드백 → 우리의 응답"* 프레임으로 열어 평가자에게 반응성·동기를 동시에 전달.

|---|---|---|---|
| 1 | **브릿지**: 중간 요약 + 받은 피드백 | 반복(최소) | **1장** — 무엇을 예측/어디까지 했나 + 교수님 피드백 핵심 → "이제 그 답을 보여드립니다" |

## 7. 예상 질의응답 (피드백 직결)

- **Q. 중간엔 VIF로 제거했는데 왜 바꿨나?** → 예측엔 공선성 사전 제거가 정보 손실. ablation로 미제거가 동등/우수임을 보였고, 트리·규제 모델은 공선성에 강건. 해석은 별도 파이프라인으로 분리.
- **Q. 그럼 VIF는 쓸모없나?** → 아니다. **해석용** 파이프라인에서 저공선성 대표 변수 선정·SHAP 안정화 근거로 여전히 사용. 목적이 다를 뿐.
- **Q. SHAP 기여 분산은 어떻게 보였나?** → Δus10y(t-1)와 한미금리차의 상관/interaction을 정량화, 군집 SHAP으로 합산 보정. permutation·drop-column과 거동 차이도 제시.
- **Q. 예측/해석 변수 집합이 다르면 결론이 흔들리지 않나?** → 두 집합 모두 "글로벌 금리 채널이 핵심"이라는 결론은 동일. 목적별 최적화일 뿐 상반되지 않음을 명시.
