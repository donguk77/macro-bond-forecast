## 1. 프로젝트 개요

### 1.1 주제
국내외 거시경제 지표를 입력으로 하여 한국 국고채 10년물 금리의 일별 변화량(`Δy`)을 **분위수 회귀(0.05/0.5/0.95)** 형태로 예측하는 다변량 시계열 딥러닝 모델을 구축한다. 단순한 점추정을 넘어 예측 구간(90%)을 제공하고, SHAP 으로 거시 변수의 시차 효과를 정량적으로 분석한다.

### 1.2 주제 분류
안내문 §2 주제 후보 7번 "전력/수요 예측" 유사 카테고리(시계열 예측). 학생 프로젝트 중 채권/금리를 다루는 사례가 드물어 포트폴리오 차별화 효과가 크다.

### 1.3 프로젝트 목표
- 문제 정의 → 데이터 → 전처리 → 모델링 → 평가 → SHAP → 시각화·문서화의 완결된 파이프라인.
- Naive · ARIMA · XGBoost 베이스라인 대비 LSTM 분위수 회귀의 성능 개선을 정량적으로 입증.
- SHAP 으로 한미 금리 동조성·인플레 시차 등 거시경제 인사이트 도출.
- AI 도구 사용과 검증 이력을 매주 누적 기록(VALIDATION_LOG 14건+)으로 남김.

## 2. 문제 정의

### 2.1 타겟 변수
한국 국고채 10년물 일별 금리의 **변화량** `Δy_{t+1} = (y_{t+1} − y_t) × 100` (단위: bp). 레벨이 아닌 변화량을 타겟으로 하는 이유는 일별 채권 레벨에서 Naive 베이스라인이 강력해 LSTM 우위 입증이 어렵기 때문이다. 변화량 + 분위수 회귀 구조는 채권 실무(VaR·리스크) 정합성도 높다.

10년물 선택 사유: 시장 대표성, 정보 풍부도, 데이터 안정성(ECOS 1995년~), 적정 변동성.

### 2.2 예측 시점 및 출력
- 1차: t+1 변화량을 분위수 [0.05, 0.5, 0.95] 동시 예측 → **점추정 1개 + 90% 예측 구간**.
- t+5 확장은 시간 여유 시 선택.

### 2.3 타겟 데이터 처리
- ECOS **지표금리(YTM)** 사용 명시.
- 만기 갱신(roll-over) 발행일 ±2일은 분기 점프(0.5~5bp)가 SHAP 해석을 오염시키므로 **제외 옵션** 적용.
- 한국 휴장일의 타겟은 **행 drop** (forward fill 시 가짜 데이터 생성 방지).

## 3. 입력 변수 설계

### 3.1 변수 선정 원칙
Box & Jenkins 의 **Parsimonious Model** 원칙을 따르되, 변수 선정 근거를 사전 이론이 아닌 **EDA 결과(VIF·Granger·상관)** 로 정당화한다. 광역 수집 후 동적 결정 방식.

### 3.2 변수 결정 시점 (안전장치 포함)

```
1주차: 광역 22개 수집 + 결측·이상치·기초 통계
2주차: 상관·VIF·단변량 Granger 분석 완료 (산출물 3건 사전 마감)
3주차 금: 잠정 freeze (8개 + 사유 문서화) + 5주차 ablation 대상 사전 명시
5주차: 사전 명시한 ablation 대상 일괄 실행
```

> ⚠️ "잠정 freeze 가 결과적 freeze 로 굳어지는" 함정을 막기 위해 안전장치 3개 운영:

**(a) 3주차 마감 산출물 강제** — 산출물 4건을 코드/표/문서로 외화:
1. 상관관계 매트릭스 (heatmap)
2. VIF 표
3. Granger 인과성 결과
4. 변수 8개 + 사유 문서화 (`VALIDATION_LOG` 와 별도 .md)

(C 수정) 일정 부담 분산을 위해 **(1)(2)(3)은 2주차 끝까지 사전 마감**, 3주차는 (4) 통합·확정 + ablation 대상 결정에 집중.

**(b) 5주차 ablation 대상 사전 명시** — 3주차에 결정. 5주차에 결정 미루기 금지.
- (C 수정) 환율 ablation 1개는 **필수**. 추가 1~2개는 **시간 여유 시 선택**으로 한정.

**(c) Ablation 음수 채택 원칙** — 결과가 부정적이어도 그 음수 자체를 결과로 보고. "왜 효과가 없었는가" 를 안내문 §9 "오류 분석 20%" 자료로 활용.

### 4.3 전처리 절차 (누수 차단 강화)

상세는 `data_leakage_checklist.md` (10개 체크 항목).

1. **시간축 동기화**: 영업일 reindex (한국·미국 영업일 합집합).
2. **월별 변수**: 발표일 시프트 후 ffill.
   - 한국 CPI: 익월 2~5일 발표 → +1개월 lag.
   - 미국 CPI: 익월 10~15일 발표 → **+1개월 + 추가 2주 lag** 또는 release-date join.
3. **정책 변수 t-1 강제**: `kr_base_rate`, `us_fed_funds` 항상 `t-1`.
4. **한국 휴장일 타겟 drop** (forward fill 시 가짜 데이터).
5. **Backward fill / 양방향 보간 절대 금지** (미래 정보 누수).
6. **정규화**: Robust Scaler **Train 만 fit**, val/cal/test transform.
7. **Lag**: `pandas.shift(k)` (k≥1).
8. **Rolling**: `.rolling(w).mean().shift(1)` 패턴 강제 (현재 시점 미포함).
9. **Look-back**: 30일 기본, 14·60일 비교.

## 5. 모델링 전략

### 5.1 베이스라인 (필수)
- **Naive (변화량)**: `Δŷ_{t+1} = 0`.
- **ARIMA**: 자기상관·추세 학습 능력 기준점.
- **XGBoost**: lag feature 기반. `objective="reg:quantileerror"` 분위수 회귀 적용.

### 5.2 메인: LSTM 분위수 회귀

다변량 LSTM. 입력 텐서 `(samples, 30 timesteps, ~8 features)`, 출력 `[q_0.05, q_0.5, q_0.95]`. 손실 **Pinball Loss(Quantile Loss)**.

```python
def pinball_loss(pred, target, q):
    diff = target - pred
    return torch.max(q * diff, (q - 1) * diff).mean()
```

게이트 구조 활용:
- Forget Gate: 과거 누적 정보 중 무의미 부분 필터링.
- Input Gate: 최근 정책금리·인플레 신호 반영 강도.
- Output Gate: 다음 영업일 변화량 3개 분위수 산출.

### 5.3 분위수 출력 후처리
**monotonicity 위반**(`q_0.05 > q_0.5` 등) 발생 시 후처리 sort 또는 quantile crossing penalty.

### 5.4 교차검증
**일반 K-fold 절대 금지** (sklearn 기본 K-fold = 미래로 과거 예측 = 누수). `TimeSeriesSplit` 또는 expanding window walk-forward 만 사용.

### 5.5 하이퍼파라미터 튜닝
**수동 grid 5×5** (학습률 5수준 × hidden 5수준). Optuna 는 시간 여유 시.

### 5.6 Conformal Prediction (선택 부록)
Coverage 90%±3%p **미달 시** Conformalized Quantile Regression(CQR) 후처리를 선택 부록으로 적용. `mapie` 라이브러리 사용. 단, **marginal coverage 보장은 점근적·근사**이며 distribution shift 에서 깨질 수 있음을 명시.

메인 평가는 raw 분위수 + sort 후처리로 진행.

### 5.7 과적합 방지
- Dropout (0.2~0.4)
- Early Stopping (patience=10)

## 6. SHAP 기반 설명 가능성 + 오류 분석

### 6.1 SHAP 적용 전략
**각 분위수(q_0.05/q_0.5/q_0.95) 모델에 SHAP 따로 적용** 후 차분 시각화.
> 분위수 폭(`q_0.95 − q_0.05`)에 SHAP 직접 적용은 efficiency·local accuracy 공리 위반 우려로 폐기.

### 6.2 SHAP 검증 일정
- **1주차**: XGBoost + TreeExplainer 로 분석 흐름 먼저 검증 (Hello World).
- **4주차**: LSTM-SHAP DeepExplainer 적용. 백업: **Permutation Importance**.

### 6.3 핵심 분석 질문
1. 미국 10년물이 한국 금리에 며칠 시차로 반영?
2. 기준금리 인상 후 시장 금리 반응 지속 기간?
3. VIX 급등의 영향 지속 기간?
4. CPI 신호의 장기금리 반영 메커니즘?
5. 위기 국면에서 어떤 변수가 q_0.95 분위수를 가장 키우는가?

### 6.4 오류 분석 4축 프레임

> 안내문 §9 "평가·오류 분석 20%" 직격.

**(a) 방향성 오답** — `sign(ŷ_0.5) ≠ sign(Δy)` top-20. 거시 이벤트와 매칭.

**(b) 큰 변동 미예측** — `|Δy| > 5bp` 인데 `|ŷ_0.5| < 1bp` top-20. 이벤트 더미로 발표일 vs 비발표일 분리 분석.

**(c) Coverage Miss** — 실제값이 90% 구간 밖. 위기 vs 정상, 보정 전후 비교, covariate shift 진단.

**(d) 위기 vs 정상 구간 SHAP 평균 차이** — 두 구간의 변수별 |SHAP| 평균 비교.

### 6.5 인과성 검토
모델이 학습한 상관관계가 거시경제학 상식과 부합하는지 확인. 예: 기준금리 상승 SHAP 양(+), VIX 상승 음(-). 어긋나면 데이터 누수·과적합 의심하고 재검증.

## 7. 평가 지표 및 목표

| 지표 | 정의 | 의미 | 목표 |
|------|------|------|------|
| **Pinball Loss** | 분위수별 비대칭 손실 | 분위수 회귀 표준 | Naive 대비 개선 (DM test 통계 유의) |
| RMSE / MAE (Δy) | 변화량 점추정 오차 | 전통적 비교 | Naive 대비 개선 (도전적; 미달 시에도 분석 가치) |
| **방향성 정확도** | 상승·하락 적중률 | 금융 실무 핵심 | **55% 이상** (메인 차별화 지표) |
| **Coverage Rate (90%)** | 90% 구간 실제 포함률 | 예측 구간 신뢰성 | raw 보고 + (선택) 보정 비교, 미달 시 §6.4(c) 분석 |
| Sharpness | 예측 구간 평균 폭 | 구간 효율성 | 정상구간 기준 비교, 위기구간 적절히 넓어질 것 |
| **DM test** | 모델 비교 통계 검정 | 유의성 검증 | HAC(Newey-West) + **HLN 소표본 보정** + Bonferroni 다중비교 |
| 위기구간 RMSE | §4.4 위기일 별도 평가 | 강건성 | 정상구간 1.5배 이내 |

**(C 수정)** v3에서 폐기했던 "Naive 대비 RMSE 10% 개선" 절대 수치는 v5에서도 **부활하지 않음**. 일별 채권에서 RMSE 10% 개선은 학술 문헌상 어려운 목표. 메인 차별화는:
1. **방향성 정확도 55%** — 금융 실무 직결.
2. **DM test 통계적 유의성** — Naive 대비 의미 있는 우위.
3. **Coverage·Sharpness 우위** — Naive 가 사실상 못 하는 영역.

## 11. 포트폴리오 차별화 포인트

1. **거시경제 도메인** — 학생 프로젝트에서 드문 채권/금리. 자산운용·증권·중앙은행 분야 어필.
2. **EDA 기반 변수 선정의 데이터 근거** — VIF·Granger·ablation 결과로 변수 선정을 정량 정당화.
3. **환율 제외 결정의 이중 근거** — 정성적 논거(정부 개입) + 정량적 근거(ablation).
4. **분위수 회귀 + 90% 예측 구간** — 점추정 너머 불확실성 정량화, 채권 실무(VaR) 정합.
5. **위기구간 정량 정의** — rolling vol + 이벤트 캘린더 OR. 서사가 아닌 코드 라벨링.
6. **이벤트 더미 4개** (FOMC·금통위·美 NFP·美 CPI) — 일별 채권 변동성의 1차 동인 명시 모델링.
7. **오류 분석 4축 프레임** — 방향성·큰 변동·Coverage·SHAP 차분.
8. **데이터 누수 차단 체크리스트** — 7개 운영. 학부 수준에서 드문 엄격성.
9. **AI 사용 기록 14건+ 누적** — 안내문 최소 3건의 4.5배.
10. **변수 freeze 안전장치 3개** — 산출물 강제·ablation 사전 명시·음수 채택 원칙으로 "잠정의 함정" 차단.

### (선택 확장) — 시간 여유 시 학술 강화

본 v5 는 **학부 7주 안정 버전**이다. 시간 여유 생기면 다음 학술 강화 항목을 추가 가능 (외부 학술 리뷰 권고):

- **Adaptive Conformal Inference (ACI)**: distribution shift 적응적 보정. 시계열 CP 의 최신 방법.
- **Walk-forward 3 fold expanding window + LSTM 재학습**: 단일 분할 hyperparam 과적합 방지.
- **CRPS (Continuous Ranked Probability Score)**: 분위수 회귀 표준 평가 지표.
- **외국인 채권 보유잔고 + 한미 스프레드(`kr10y - us10y`)** 추가: 채권 도메인 1차 동인.
- **SHAP 인터랙션 효과**: 변수쌍 상호작용 분석.

## 12. 결론

본 프로젝트는 안내문이 요구하는 7주·2~3인 팀의 현실적 제약 안에서 **시계열 예측·딥러닝·XAI** 핵심 ML 역량을 모두 다루면서, 거시경제라는 차별화된 도메인을 결합한 균형 잡힌 설계이다. 한국은행 ECOS 와 FRED 의 무료 API 로 데이터 접근성을 확보했고, Naive 부터 LSTM 분위수 회귀·SHAP 까지 단계적 모델링으로 학습 곡선을 관리한다.

**v5 의 핵심은 변수 선정 시점의 균형**이다. v4 의 1주차 즉시 freeze 는 EDA 를 형식적으로 만들 위험이 있었고, v3 의 동적 결정만으로는 일정이 임계경로에 부딪힐 위험이 있었다. v5 는 v4 의 단순화 기조(TCN 폐기·CQR 강등·수동 grid·Streamlit 선택) 위에 변수 freeze 시점만 v3 의 3주차 잠정 freeze + 5주차 ablation 구조로 회귀시키되, **안전장치 3개**(3주차 산출물 강제·ablation 대상 사전 명시·음수 채택 원칙)를 신설하여 "잠정 freeze 가 결과적 freeze 로 굳어지는" 함정을 차단했다.

v5 마크다운에는 **C 미세 수정 5개**를 추가 적용했다: 3주차 부담 분산(상관·VIF·Granger 산출물 2주차 사전 마감), 5주차 ablation "환율 필수 + 추가 선택" 한정 명시, "Naive 대비 10% 개선" 절대 수치 폐기 후 방향성·DM test·Coverage 우위로 차별화 재정의, §3.3 변수 표 비고 컬럼 복원, §11 끝 "선택 확장 (학술 강화)" 절 신설로 향후 확장 여지 명시.

이러한 설계는 안내문의 핵심 원칙 "AI 도구를 활용하되 결과의 타당성을 스스로 검증하는 습관" 을 매주 누적되는 14건+ 검증 기록으로 충실히 따르는 사례가 될 것이다.
