## Page 5: 베이스라인 모델 비교 — 막대그래프 → 지표 비교 표로 변경

### 현재 문제

- VAL RMSE (7.10 vs 7.11), TEST RMSE (4.65 vs 4.65) 차이가 거의 없어서 막대그래프가 의미 없어 보임
- 베이스라인 비교 목적에 맞게 여러 지표를 함께 보여주는 게 더 적절

### 수정 방향

막대그래프 대신 **Naive와 ARIMA의 다양한 지표를 비교하는 표**로 교체.
베이스라인 간 성능이 거의 동일하다는 것 자체가 핵심 메시지 — "단순 시계열로는 금리 변화 예측이 어렵다"

### 수정 후 내용

| 지표           | Naive (Δŷ=0)  | ARIMA(1,0,1) |
| -------------- | --------------- | ------------ |
| Test RMSE (bp) | **4.647** | 4.651        |
| Test MAE (bp)  | **3.477** | 3.483        |
| 방향 정확도    | ~50%            | 49.7%        |

**핵심 메시지:**

> Naive와 ARIMA가 거의 동일한 성능 → 단순 시계열 구조(자기상관)만으로는 금리 변화 예측이 어려움 → 다변량 거시경제 변수 모델(XGBoost, LSTM)이 필요한 근거

## Page 6: XGBoost 분위수 회귀 — Q&A 대비 추가

### 현재 문제

- XGBoost 구조 설명과 차트는 있으나, 분위수 예측의 실용적 의미 설명이 부족
- "구간이 대략 ±8bp로 일정한데 의미가 있나?" 질문 예상

### XGBoost v2 예측 구간 차트 (✅ 생성 완료)

`reports/no_leak_v2/figures/xgb_v2_pred_interval_fold3.png` — fold3 (2023-2025) CQR 보정 후 90% 예측 구간

### Q&A 대비 — 발표 시 또는 슬라이드 하단에 추가

**Q: "예측 구간이 ±8bp로 거의 일정한데 의미가 있나?"**

A:

1. 일정하지 않음 — 변동성 높은 구간(2023 초, 2024-09)에서 구간이 넓어지고, 안정기에 좁아짐. XGBoost가 시장 상황에 따라 불확실성을 조절하고 있음 (LSTM v1의 완전 고정 구간과 비교)
2. ±8bp 자체가 실무적으로 유의미 — "내일 금리가 -8bp~+8bp 사이에 90% 확률로 들어온다"는 정보는 VaR 한도 설정에 직접 사용
3. 점 예측과의 차이 — Naive는 "변화 0" 점 하나만 제공하지만, 분위수 회귀는 최악(q05)/최선(q95) 시나리오를 동시에 제공. 리스크 관리에서 평균이 아닌 꼬리(tail)가 중요

## Page 7: LSTM 분위수 회귀 — 오타 수정 + 차트를 v2 버전으로 교체

### 오타 수정

- "3개의 출력에 각각 **0.5, 0.5, 0.95** 분위수에 수렴하도록 학습"
- → "3개의 출력에 각각 **0.05, 0.5, 0.95** 분위수에 수렴하도록 학습"

### 차트 교체 (✅ v2 차트 생성 완료)

현재 차트는 **LSTM v1** (누수 수정 직후, dir_acc 49.8%) — q50이 일직선이고 구간이 고정 폭.
→ 아래 **LSTM v2** 차트로 교체:

- **예측 구간 차트**: `reports/no_leak_v2/figures/lstm_v2_pred_interval_fold3.png` (fold3: 2023-2025)
- **시드별 dir_acc 비교**: `reports/no_leak_v2/figures/lstm_v2_dir_acc_by_seed.png`

### LSTM v1 → v2 학습 과정 설명 보완

**v1 (기본 8개 변수)이 왜 안 됐나:**

- 5×5 grid search (lr × hidden_dim) → 25개 조합 전부 dir_acc ≈ 49.5%
- 누수 수정(shift(1)) 후 미국 변수 당일 정보 차단 → 가장 강력한 신호 소실
- ACF에서 Δy가 백색잡음에 가까움 → 순차 학습 강점 발휘 안 됨
- 8개 원본 변수만으로는 변수 간 관계(차이, 비율)를 자동 학습 못함

**v2 (파생변수 5개 추가) 후 회복:**

- 5개 파생변수(spread, delta, crisis_dummy) 추가
- LSTM v2 3-fold 평균 **59.2%**, fold3에서 **66.1%**까지 회복
- 하지만 XGBoost v2 (61.78% pooled)가 전체적으로 우세 → **XGBoost를 메인 모델로 채택**

> 발표 포인트: "LSTM이 안 됐다"가 아니라 **"누수를 정직하게 수정하고, 원인을 분석하고, 파생변수로 회복시킨 과정 자체가 프로젝트의 가치"**

## Page 10: 최종 결과 비교 — LSTM v1 → v2로 교체 + 전체 지표 추가

### 현재 문제

- LSTM v1 (49.8%)만 표시되어 있어서, LSTM이 그냥 실패한 것처럼 보임
- XGBoost v2 Test RMSE, Coverage, Sharpness 빠져 있음

### 수정 후 표 (✅ 3-fold walkforward 실측치 반영 완료)

| 지표                       | Naive | ARIMA | LSTM v2                        | XGBoost v2 (CQR)               |
| -------------------------- | ----- | ----- | ------------------------------ | ------------------------------ |
| Test RMSE (bp, 3-fold avg) | 4.67  | 4.65  | **4.49**                 | **4.56**                 |
| 방향 정확도 (3-fold 평균)  | 50%   | 49.7% | **59.2% ± 6.9%**        | **61.0% ± 2.8%**        |
| 방향 정확도 (pooled)       | -     | -     | **61.6%**                | **61.8%**                |
| DM 검정 vs Naive (pooled)  | -     | -     | **DM=-9.83, p<0.001 ✓** | **DM=-8.78, p<0.001 ✓** |
| Coverage 90% (3-fold avg)  | -     | -     | **77.3%**                | **87.3%** (CQR)          |
| Sharpness (bp, 3-fold avg) | -     | -     | **9.50**                 | **15.64** (CQR)          |

> **수치 출처**: `reports/no_leak_v2/walkforward_lstm_v2_full.csv`, `walkforward_xgb_v2_full.csv`, `walkforward_lstm_dm_v2.csv`
> **차트 파일**: `reports/no_leak_v2/figures/model_comparison_summary_v2.png` (요약 테이블), `xgb_vs_lstm_v2_comparison.png` (3-panel 비교)

### 이 표에서 읽어야 할 핵심 메시지

1. **두 모델 모두 Naive 대비 통계적 우위** — DM 검정 pooled p<0.001 (Bonferroni 통과)
2. **XGBoost v2가 메인 모델** — 3-fold 평균 61.0% ± 2.8%로 안정적, LSTM v2는 59.2% ± 6.9%로 편차가 큼
3. **LSTM v2는 RMSE에서 우위** — 3-fold avg RMSE 4.49 bp vs XGBoost 4.56 bp (점 예측 정확도는 LSTM이 살짝 나음)
4. **Coverage**: XGBoost CQR (87.3%) >> LSTM raw (77.3%) — CQR 후처리가 구간 예측을 크게 개선
5. **Sharpness**: LSTM (9.50 bp) << XGBoost CQR (15.64 bp) — LSTM 구간이 좁지만 Coverage도 낮음. XGBoost CQR은 과보장 경향
6. **fold별 패턴**: fold1(COVID)은 두 모델 모두 ~50-59% (distribution shift), fold3(안정기)에서 LSTM이 66.1%로 XGBoost(64.2%) 상회

## 신규 슬라이드: EMH — 61.1%가 왜 의미 있는가

### 현재 문제

중간 발표에서 설명했던 효율적 시장 가설(EMH) 근거가 완전히 빠져 있음. 결과표에 61.1%만 있으면 "겨우 61%?" 반응이 나올 수 있음.

### 추가 슬라이드 내용

**제목: 방향 정확도 61%의 의미**

- 효율적 시장 가설(EMH): 모든 공개 정보가 이미 가격에 반영 → 누구도 50%보다 의미 있게 잘 맞히기 어려움
- 학술 논문 합격선: **52~53%만 통계적으로 유의해도 인정**
- 헤지펀드가 ML에 거액을 투자하는 이유 = 2~3%p의 우위
- 본 연구 결과: **61.1% + DM 검정 p < 0.0001** → 우연이 아님을 통계적으로 입증
- Bootstrap 95% CI: **[59.3%, 64.2%]** → 하한이 학술 합격선 53%를 명확히 상회
- 베이스라인 대비 **+11%p** 우위

### 추가 슬라이드 내용

**제목: 오류 분석 — 모델은 언제, 왜 틀리는가?**

| 분석 축        | 내용                                 | 발견                                                                  |
| -------------- | ------------------------------------ | --------------------------------------------------------------------- |
| 방향성 오답    | 부호를 틀린 사례의 공통점            | 금리 모멘텀 전환점에 집중                                             |
| 큰 변동 미예측 | \|Δy\|>5bp인데 모델은 1bp 이내 예측 | FOMC, 한은 금리 결정일에 발생                                         |
| Coverage Miss  | 90% 구간 이탈 시점 분석              | fold2(금리 인상기) XGB 70.1%, LSTM 66.0%로 급락 — distribution shift |

**핵심 결론:**

> 모델은 안정적 시장에서 유효하나, 레짐 전환(regime shift) 구간에서 한계.
> 이는 학습 데이터에 없는 패턴이 등장하는 본질적 문제.
