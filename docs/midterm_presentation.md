## 슬라이드 2. 연구 배경 & 선행연구 차별점 (1분 30초)

### PPT 내용

**관련 선행연구:**

| 논문                             | 내용                                                                                   | 본 연구와의 차이                                 |
| -------------------------------- | -------------------------------------------------------------------------------------- | ------------------------------------------------ |
| Koenker & Bassett (1978)         | 분위수 회귀(Quantile Regression) 원조 논문. 비대칭 손실 함수로 조건부 분위수 직접 추정 | 이론적 기반. 본 연구는 이를 XGBoost·LSTM에 적용 |
| Taylor (2000)                    | 신경망에 Pinball Loss를 적용한 분위수 회귀 최초 제안                                   | LSTM 분위수 회귀의 직접적 근거                   |
| Romano et al. (2019)             | Conformalized Quantile Regression(CQR) — 분포 가정 없이 예측 구간 커버리지 보정       | 본 연구의 CQR 보정 방법론의 출처                 |
| Diebold & Mariano (1995)         | 두 예측 모델의 정확도 차이 통계 검정 (DM test)                                         | 모델 비교 유의성 검정에 사용                     |
| Caballero & Krishnamurthy (2009) | EM(신흥시장) 자본 흐름에서 금리차가 1차 동인임을 실증                                  | v2 파생변수 `spread_10y_t1`의 도메인 근거      |
| Chen & Yang (2004)               | 거시 변수 기반 채권 금리 예측, 점 추정 + RMSE 평가                                     | 본 연구는 분위수 + 방향성 + DM 검정으로 확장     |

**기존 연구 vs 본 연구 비교표:**

| 구분      | 기존 연구                       | 본 연구                                               |
| --------- | ------------------------------- | ----------------------------------------------------- |
| 예측 대상 | 금리 수준(level) 또는 월별 방향 | **일별 변화량(Δy)** — 더 어려운 문제          |
| 출력      | 점 예측 1개 (평균)              | **분위수 3개 (q05, q50, q95)** — 불확실성 추정 |
| 구간 보정 | 없음                            | **CQR (Romano et al. 2019)** — 커버리지 보정   |
| 누수 검증 | 대부분 미검증                   | **시차 누수 발견 & 수정** — 정직한 성능 보고   |
| 통계 검증 | RMSE 비교만                     | **DM 검정 + Walk-forward 3-fold**               |

### 대본

> 먼저 기존 연구와의 차별점을 말씀드리겠습니다. 저희가 사용한 방법론은 여러 선행연구에 기반합니다. 분위수 회귀는 Koenker와 Bassett가 1978년에 제안한 표준 기법이고, 이를 신경망에 적용한 것은 Taylor가 2000년에 확립했습니다. 예측 구간 보정에는 Romano 등이 2019년에 제안한 CQR을 사용했고, 모델 비교에는 Diebold-Mariano 검정을 적용했습니다.
>
> 기존 채권 금리 예측 연구들과 비교하면, 저희 연구는 세 가지가 다릅니다.
>
> 첫째, 예측 대상이 일별 변화량입니다. 노이즈가 크기 때문에 훨씬 어려운 문제입니다.
>
> 둘째, 분위수 회귀를 적용해서 "내일 금리가 얼마나 변할지"뿐 아니라 "최악의 경우 얼마까지 변할 수 있는지"를 함께 추정합니다. 이건 리스크 관리 측면에서 실무적 가치가 있습니다.
>
> 셋째, 프로젝트 진행 중 데이터 누수를 직접 발견하고 수정하는 과정을 거쳤습니다. 대부분의 연구에서 간과하는 부분인데, 이 부분은 뒤에서 자세히 설명드리겠습니다.

## 슬라이드 4. 데이터 분할 & 전처리 (1분)

### PPT 내용 — 분할 다이어그램

```
├─ Train (2010-2020) ──┤─ Cal (2021) ─┤─ Val (2022) ─┤─ Test (2023-2025) ─┤
    모델 학습 (~2,500일)    CQR 보정      하이퍼파라미터       최종 평가
                            전용           튜닝 전용          (학습에 미사용)
```

**전처리 파이프라인:**

- 변환: 정책변수 shift(1), 전 변수 diff().shift(1)
- Lag 피처: 1, 5, 10, 20, 30일
- Rolling 피처: 5일, 20일 평균 & 표준편차
- 스케일링: RobustScaler (Train 구간만 fit)
- 최종: 8개 변수 → **108개 피처**

### 대본

> 데이터는 2010년부터 2025년까지 약 3,700 거래일입니다. 4구간으로 나눴는데, Train은 모델 학습용, Cal은 CQR 보정 전용, Val은 하이퍼파라미터 튜닝용, 그리고 Test는 최종 평가용으로 학습에 절대 사용하지 않았습니다.
>
> 전처리는 각 변수에 대해 Lag 피처 5개와 Rolling 통계를 생성해서 8개 원본 변수를 108개 피처로 확장했습니다. 스케일링은 RobustScaler를 썼고, 중요한 건 Train 구간 통계로만 fit해서 미래 정보 누수를 차단한 점입니다.

## 슬라이드 5. 베이스라인 모델 (1분)

### PPT 내용

**Naive 모델:**

```
예측: Δŷ_{t+1} = 0 (내일도 변화 없음)
원리: 일별 금리 변화량의 장기 평균 ≈ 0 → 가장 안전한 점 예측
특징: 학습 없음, 파라미터 없음 — 모든 ML 모델이 이겨야 할 최소 기준선
      효율적 시장 가설(EMH)이 성립하면 이것이 최적
```

**ARIMA(1,0,1) 모델:**

```
수식: Δŷ_t = c + φ₁·Δy_{t-1} + θ₁·ε_{t-1} + ε_t
      AR(1): 어제 변화량의 관성 → φ₁ = -0.178
      MA(1): 어제 충격의 잔향  → θ₁ = +0.189
결과: φ₁과 θ₁이 거의 상쇄 (-0.178 + 0.189 ≈ 0) → 사실상 Naive와 동일
의미: 단변량 시계열 자기상관만으로는 금리 변화를 예측할 수 없음
      → 다변량 거시 변수 모델(XGBoost, LSTM)이 필요한 근거
```

**결과 비교:**

| 모델                   | 입력 변수       | 학습             | Test RMSE (bp) | Test MAE (bp) | 방향 정확도 |
| ---------------------- | --------------- | ---------------- | -------------- | ------------- | ----------- |
| **Naive**        | 없음            | 없음             | 4.647          | 3.477         | 50.0%       |
| **ARIMA(1,0,1)** | Δy 자기 자신만 | AR·MA 계수 추정 | 4.651          | 3.483         | 49.7%       |

### 대본

> 베이스라인으로 Naive 모델과 ARIMA를 설정했습니다.
>
> Naive는 "내일도 변화 없다"고 예측하는 가장 단순한 모델입니다. 일별 금리 변화의 장기 평균이 거의 0이기 때문에, 효율적 시장 가설이 맞다면 이것이 최적 예측이 됩니다. 학습도 파라미터도 없지만 의외로 강력한 기준선입니다.
>
> ARIMA(1,0,1)은 시계열 표준 모델입니다. AR 항은 어제 변화량의 관성을, MA 항은 어제 충격의 잔향을 모델링합니다. 그런데 추정된 AR 계수 -0.178과 MA 계수 +0.189가 거의 상쇄되어서 사실상 0에 가까운 예측을 합니다. 즉 단변량 시계열 구조만으로는 금리 변화를 예측할 수 없다는 뜻이고, 이것이 다변량 거시 변수를 쓰는 XGBoost나 LSTM이 필요한 이유입니다.
>
> 둘 다 방향 정확도가 약 50%, 동전 던지기 수준입니다. 이 50%가 저희 모델이 이겨야 할 기준선입니다.

## 슬라이드 6. XGBoost 분위수 회귀 (1분)

### PPT 내용

**XGBoost 작동 원리:**

```
1단계: 초기 예측 F₀ = 전체 평균
2단계: 잔차(residual) 계산 — "1단계에서 못 맞춘 부분"
3단계: 작은 결정 트리가 잔차를 예측
4단계: 기존 예측 + 새 트리 예측 → 업데이트
→ 2~4단계를 수백 번 반복 (순차 보정)
= "약한 학습기의 연속적 보완"
```

**일반 XGBoost vs 분위수 XGBoost:**

| 구분      | 일반 XGBoost         | 분위수 XGBoost                               |
| --------- | -------------------- | -------------------------------------------- |
| 손실 함수 | MSE (평균 오차 제곱) | **Pinball Loss** (비대칭 손실)         |
| 출력      | 평균값 1개           | **분위수별 값**                        |
| 모델 수   | 1개                  | **3개 (q05, q50, q95 각각 따로 학습)** |

**Pinball Loss 직관:**

```
Pinball Loss(q, y, ŷ) = 
   q × (y - ŷ)       y > ŷ일 때 (실제 > 예측 = 과소예측)
   (1-q) × (ŷ - y)   y < ŷ일 때 (실제 < 예측 = 과대예측)

q = 0.05 → 과소예측 벌점  5%, 과대예측 벌점 95% → 낮은 값 쪽으로 학습
q = 0.50 → 과소예측 벌점 50%, 과대예측 벌점 50% → 중앙값 학습
q = 0.95 → 과소예측 벌점 95%, 과대예측 벌점  5% → 높은 값 쪽으로 학습
```

→ [q05, q95] 구간 = **90% 예측 구간** 자동 생성

**본 연구 XGBoost 하이퍼파라미터:**

| 항목          | 값                                                 |
| ------------- | -------------------------------------------------- |
| max_depth     | 4 (grid search 최적값)                             |
| n_estimators  | 500 상한, early stopping 적용 (실제 53~215 라운드) |
| learning_rate | 0.03 (q05) / 0.05 (q50) / 0.10 (q95)               |
| 입력 피처 수  | 108개 (8변수 × lag/rolling) + 파생변수 5개        |

### 대본

> XGBoost에 분위수 회귀를 적용한 방법을 설명드리겠습니다. XGBoost는 약한 결정 트리를 순차적으로 추가하면서 이전 모델이 못 맞춘 잔차를 보정하는 부스팅 알고리즘입니다.
>
> 핵심은 손실 함수입니다. 일반 XGBoost는 MSE를 최소화해서 평균값을 예측하지만, 저희는 Pinball Loss라는 비대칭 손실 함수를 사용했습니다. q=0.05로 학습하면 과대예측에 95%의 벌점을 줘서 모델이 낮은 값 쪽으로 학습됩니다. 반대로 q=0.95면 과소예측에 95% 벌점을 줘서 높은 쪽으로 학습되고요.
>
> 이렇게 q05, q50, q95 세 모델을 따로 학습하면 90% 예측 구간이 자동으로 만들어집니다. 트리 깊이 4(grid search 최적값), 최대 500그루에 early stopping을 적용해서 과적합을 방지했습니다.

## 슬라이드 7. LSTM 분위수 회귀 — 원리 (1분 30초)

### PPT 내용

**LSTM 게이트 구조 (핵심 3개):**

```
Forget Gate: "과거 정보 중 무엇을 잊을까?"
  f_t = σ(W_f · [h_{t-1}, x_t])     → 0~1 사이 비율 출력
  위기 구간에서 평상시 패턴을 빠르게 잊고 전환 가능

Input Gate: "새 정보 중 무엇을 기억할까?"
  i_t = σ(W_i · [h_{t-1}, x_t])     → 저장 비율
  g_t = tanh(W_g · [h_{t-1}, x_t])  → 후보 새 정보

Cell State 갱신:
  c_t = f_t × c_{t-1} + i_t × g_t   → "잊을 건 잊고, 새로 받을 건 받기"
  (덧셈 구조 → 기울기 소실 방지 → 장기 의존성 학습 가능)

Output Gate: "무엇을 출력할까?"
  h_t = o_t × tanh(c_t)             → 다음 시점으로 전달
```

**전체 모델 구조:**

```
입력: (batch, 30일 시퀀스, 8변수)
         ↓
   LSTM 2층 (hidden=128*, dropout=0.3)
   → 30일치 거시 변수를 순서대로 읽으며 128차원 벡터로 압축
         ↓
   마지막 시점 hidden state h_T (128차원)
         ↓
   Linear 층 (128 → 3)

* 초기 모델은 hidden=64, 5주차 grid search에서 hidden=128이 최적으로 선택됨
         ↓
출력: [q05, q50, q95]  ← 숫자 3개 동시 출력
```

**핵심: 분위수는 구조가 아니라 손실 함수가 결정한다**

```python
loss = pinball(output[0], y, q=0.05)   # 첫 번째 출력 → q05로 학습
     + pinball(output[1], y, q=0.50)   # 두 번째 출력 → q50로 학습
     + pinball(output[2], y, q=0.95)   # 세 번째 출력 → q95로 학습
```

LSTM 자체는 그냥 숫자 3개를 출력하는 신경망. Pinball Loss가 각 출력을 해당 분위수에 수렴하게 만듦.

**XGBoost vs LSTM 구조 비교:**

| 구분        | XGBoost                      | LSTM                                               |
| ----------- | ---------------------------- | -------------------------------------------------- |
| 분위수 방식 | **모델 3개** 따로 학습 | **모델 1개**가 3개 동시 출력                 |
| 손실 함수   | Pinball Loss (동일)          | Pinball Loss (동일)                                |
| 입력 형태   | 피처 벡터 1개 (108차원)      | **30일 시계열** (30×8)                      |
| 시간 의존성 | lag 피처로 간접 표현         | 게이트로**직접 학습**                        |
| 파라미터 수 | 트리 구조 (과적합 방어)      | **~52K** (hidden=128 기준, 과적합 위험 있음) |
| 학습 시간   | 분 단위                      | 시간 단위                                          |
| 해석성      | SHAP TreeExplainer (정확)    | SHAP DeepExplainer (근사)                          |

> 참고: Quantile Regression — Koenker & Bassett (1978), 신경망 적용 — Taylor (2000)

### 대본

> 교수님께서 이전에 LSTM이 어떻게 분위수를 출력하는지 질문하셨는데, 이 부분을 설명드리겠습니다.
>
> 먼저 LSTM의 핵심 구조입니다. LSTM은 세 개의 게이트로 구성됩니다. Forget Gate는 과거 정보 중 무엇을 잊을지, Input Gate는 새 정보 중 무엇을 기억할지, Output Gate는 무엇을 출력할지를 학습합니다. 중요한 건 Cell State가 덧셈으로 갱신된다는 점인데, 이 구조 덕분에 일반 RNN의 기울기 소실 문제 없이 30일 같은 장기 의존성을 학습할 수 있습니다.
>
> 전체 구조는, 30일치 8개 변수 시계열을 LSTM 2층에 통과시켜 128차원 벡터로 압축하고, 마지막 Linear 층에서 숫자 3개를 출력합니다. LSTM 자체는 특별한 게 없습니다.
>
> 핵심은 손실 함수입니다. 학습할 때 세 출력 각각에 다른 분위수의 Pinball Loss를 적용합니다. 첫 번째 출력에는 q=0.05, 두 번째에는 q=0.50, 세 번째에는 q=0.95의 Pinball Loss를 걸어주면, 각 출력이 해당 분위수에 수렴하도록 학습됩니다. 즉 분위수를 만드는 건 모델 구조가 아니라 손실 함수입니다. 이건 Koenker와 Bassett가 1978년에 제안한 분위수 회귀의 표준 접근법을 신경망에 적용한 것입니다.
>
> XGBoost와의 차이는, XGBoost는 분위수마다 별도 모델을 학습하지만 LSTM은 하나의 모델이 세 분위수를 동시에 학습한다는 점입니다. 또 XGBoost는 lag 피처로 시간 의존성을 간접 표현하지만, LSTM은 게이트 구조로 직접 학습합니다.

## 슬라이드 8. 데이터 누수 발견 & 수정 (1분 30초)

### PPT 내용

**문제: 한미 시장 시차 — 시간대별 상세**

```
KST 기준 하루의 시간 흐름:

09:00  한국 채권 시장 개장
```

- 방향 정확도 65.2% → 49.8% = **15.4%p 하락**
- 즉 v0에서 맞춘 예측의 상당 부분이 "미국 시장이 이미 방향을 알려줬기 때문"
- **"좋은 결과가 나왔을 때 의심하라. 높은 정확도 자체가 누수의 증거일 수 있다."**

### 대본

> 이 부분이 저희 프로젝트에서 가장 중요한 발견입니다.
>

>
> 이게 구체적으로 어떤 문제냐면, 예를 들어 3월 15일 한국 채권 금리를 예측할 때 3월 15일 미국 국채 금리를 입력으로 사용하면, 모델은 한국 장 마감 이후에 형성된 미국 시장 정보를 미리 보는 셈입니다.
>
> 수정은 간단합니다. 미국 변수 5개에 하루 시차를 적용해서, 3월 15일 한국 금리를 예측할 때는 3월 14일 미국 데이터만 사용하도록 했습니다. 그런데 수정하자마자 LSTM 방향 정확도가 65%에서 49.8%로 떨어졌습니다. 동전 던지기 수준입니다. 65%라는 성능의 대부분이 사실 누수 덕분이었던 겁니다.
>
> 이후 파생변수를 추가한 v2에서 XGBoost가 61.1%로 정직하게 회복했습니다. 이 경험이 저희에게 중요한 교훈을 줬습니다. 좋은 결과가 나왔을 때 오히려 의심해야 한다는 것, 그리고 높은 정확도 자체가 누수의 증거일 수 있다는 점입니다.

**추가된 파생변수 5개 (모두 shift(1) 적용, 누수 없음):**

| 변수           | 정의                  | 도메인 근거                                                       |
| -------------- | --------------------- | ----------------------------------------------------------------- |
| spread_10y_t1  | (us10y − kr10y)[t-1] | 한미 금리차 → 외국인 자금 흐름 (Caballero & Krishnamurthy, 2009) |
| delta_us10y_t1 | Δus10y[t-1]          | 미국 금리 변화 모멘텀 → 다음 날 한국 시초가 갭 영향              |
| delta_vix_t1   | Δvix[t-1]            | 위험회피 모멘텀 → 안전자산 선호 시차 효과                        |
| delta_dxy_t1   | Δdxy[t-1]            | 달러 강세 모멘텀 → EM 자본유출 시차 효과                         |
| crisis_dummy   | vol > 80%ile [t-1]    | 위기 구간 식별 (train-only 임계값으로 누수 차단)                  |

**파생변수를 추가한 이유:**

- XGBoost는 트리 기반 → 변수 간 **차이·비율 같은 관계를 자동으로 계산하지 못함**
- us10y와 kr10y를 각각 줘도 "금리차"라는 개념을 스스로 발견하기 어려움
- 도메인 지식으로 관계를 명시적으로 만들어 피처로 제공

**최종 결과 비교:**

| 지표                      | Naive | ARIMA | LSTM v1 | **XGBoost v2**      |
| ------------------------- | ----- | ----- | ------- | ------------------------- |
| Test RMSE (bp)            | 4.647 | 4.651 | 4.62    | —                        |
| 방향 정확도 (단일 분할)   | 50.0% | 49.7% | 49.8%   | **61.1%**           |
| 방향 정확도 (3-fold 평균) | —    | —    | —      | **61.0%**           |
| DM 검정 (vs Naive)        | —    | —    | 비유의  | **유의 (p < 0.05)** |

### 대본

> 누수 수정 후 성능 회복을 위해 도메인 지식 기반으로 파생변수 5개를 추가했습니다. 한미 금리차, 미국 금리와 VIX, 달러 인덱스의 변화량, 그리고 변동성 기반 위기 더미입니다. 모든 변수에 shift(1)을 적용해서 누수가 없음을 확인했습니다.
>
> 이 파생변수를 추가한 핵심 이유는, XGBoost가 트리 기반 모델이라 변수 간 차이나 비율 같은 관계를 자동으로 계산하지 못하기 때문입니다. "한미 금리차"라는 개념을 명시적으로 피처로 만들어줘야 합니다.
>
> 결과적으로 XGBoost v2 모델이 방향 정확도 61.1%를 달성했습니다. Walk-forward 3-fold 교차검증에서도 평균 61.0%로 일관되고, Diebold-Mariano 검정에서 Naive 대비 통계적으로 유의한 차이를 확인했습니다. 누수 없이 정직하게 베이스라인을 11%p 이긴 결과입니다.

## 슬라이드 13. (부록) 백테스트 — Walk-forward 3-fold 검증

### PPT 내용

**3개 시장 국면에서의 전략 성과 (차트 삽입: `figures/08_backtest_walkforward.png`):**

| Fold   | 기간    | 시장 상황    | S0 Buy-Hold | S1 Sign(q50)     | S2 ConfFilter    | 방향 정확도 |
| ------ | ------- | ------------ | ----------- | ---------------- | ---------------- | ----------- |
| fold1  | 2020    | COVID 급변동 | +1.3%       | **-5.6%**  | **-2.4%**  | 56.4%       |
| fold2  | 2021-22 | 금리 인상기  | -10.4%      | **-9.6%**  | **-0.8%**  | 53.5%       |
| fold3  | 2023-25 | 안정기       | +12.4%      | **+21.8%** | **+13.0%** | 61.9%       |
| POOLED | 전체    | —           | +3.3%       | +6.7%            | **+9.8%**  | —          |

**핵심 인사이트:**

- fold1(COVID), fold2(금리인상)에서 대부분 전략 손실 → **위기 구간에서의 모델 한계**
- fold3(안정기)에서만 유의미한 수익 → 시장이 안정적일 때 모델이 작동
- S2는 모든 fold에서 **손실을 최소화** — fold1: -2.4%, fold2: -0.8%
- "모델이 수익을 만들 수 있지만, 위기에서는 한계가 있다"는 **정직한 결론**

### 대본

> Walk-forward 3-fold 결과입니다. fold1은 2020년 코로나 급변동기, fold2는 2021-22년 금리 인상기, fold3은 2023-25년 상대적 안정기입니다.
>
> 솔직히 말씀드리면, fold1과 fold2에서는 대부분의 전략이 손실을 냈습니다. 모델의 방향 정확도가 53~56%로 낮아져서 거래비용을 커버하지 못한 겁니다. 수익은 fold3 안정기에 집중되어 있습니다.
>
> 하지만 여기서도 S2 Confidence Filter는 방어적입니다. fold1에서 -2.4%, fold2에서 -0.8%로 손실을 최소화했습니다. 확신 없을 때 거래하지 않으니까 위기에서 덜 잃는 겁니다.
>
> 결론적으로 모델이 수익을 만들 수 있지만 위기 구간에서는 한계가 있다는 것이 정직한 결과입니다.

## 부록: 예상 질문 & 답변

### Q1. "왜 LSTM이 아니라 XGBoost가 최종 모델인가?"

> 누수 수정 후 LSTM은 방향 정확도가 49.8%로 떨어졌습니다. 학습 데이터가 약 2,500일(10년)로 LSTM 같은 딥러닝 모델(52K 파라미터)이 일반화하기에는 부족합니다. XGBoost는 트리 기반이라 과적합 위험이 낮고, 파생변수를 명시적으로 줄 수 있어서 이 데이터 규모에서 더 적합했습니다.

### Q2. "61%면 충분한 건가?"

> 일별 금리 변화 예측에서 61%는 상당히 의미 있는 수치입니다. 첫째, 베이스라인(50%) 대비 11%p 초과이고 DM 검정으로 통계적 유의성을 확인했습니다. 둘째, 백테스트에서 61% 방향 정확도가 실제 수익으로 전환될 수 있음을 확인했습니다. 금융 시계열에서 Random Walk를 유의하게 이기는 것 자체가 도전적인 목표입니다.

### Q3. "CQR가 뭔가?"

> Conformal Quantile Regression입니다. 모델이 출력한 q05, q95 구간이 실제로 90%를 커버하는지 보장이 없기 때문에, Cal 구간(2021년)의 실제 오차를 측정해서 구간 폭을 보정합니다. Romano et al. (2019)이 제안한 방법으로, 모델에 무관하게 적용 가능한 표준 기법입니다.

### Q4. "더 복잡한 모델(CNN, Transformer)은 안 했나?"

> LSTM(52K 파라미터)도 이 데이터 규모에서 과적합했습니다. 더 큰 모델은 상황을 악화시킬 가능성이 높습니다. 데이터가 제한적인 금융 시계열에서는 모델 복잡도를 높이는 것보다 피처 엔지니어링(파생변수)이 더 효과적이었고, 실제로 XGBoost + 파생변수 조합이 이를 증명했습니다.

### Q5. "파생변수 추가가 사후 끼워맞추기 아닌가?"

> 아닙니다. 세 가지 근거가 있습니다. 첫째, 모든 파생변수는 도메인 논문에 근거합니다 (한미 금리차 → Caballero & Krishnamurthy 2009 등). 둘째, 변수 정의를 결과 확인 전에 사전 기록했습니다 (docs/features_v2_justification.md). 셋째, Walk-forward 3-fold에서 일관된 성능(61.0%)을 보여 과적합이 아님을 확인했습니다.

### Q6. "누수를 어떻게 발견했나?"

> SHAP 분석에서 us_treasury_10y의 중요도가 비정상적으로 높게 나왔고, 채널 검증에서 미국 금리가 한국 금리에 즉시(t=0) 영향을 준다는 결과가 나왔습니다. 하지만 실제로는 미국 장 마감이 한국 장 마감 14시간 후라 불가능한 인과관계입니다. 이 모순에서 시차 누수를 의심하게 됐습니다.

### Q7. "왜 환율을 입력에서 뺐나?"

> 한국 원/달러 환율은 정부 개입 구조가 강합니다 (외환당국 1,500원 방어선, 국민연금 외환스왑 등). 학습하면 거시 신호가 아닌 정책 신호를 학습하게 됩니다. 환율의 정보는 DXY(달러 인덱스)로 대체했습니다.

### Q8. "왜 변화량(Δy)을 타겟으로 잡았나?"

> 두 가지 이유입니다. 첫째, 금리 레벨은 비정상(non-stationary) 시계열이라 통계적으로 불안정합니다. 변화량은 정상 시계열이라 모델 학습이 안정적입니다. 둘째, 레벨 예측에서는 Naive(어제=오늘)가 너무 강해 ML 모델의 우위를 입증하기 어렵습니다.
